SVG’s — How do they work?

Scalable Vector Graphics, also known as SVG is a graphics format based on XML. They don’t loose quality when scaled, unlike pixel-based graphics. This makes it a good choice for graphics that need to be scaled often, like the boats on the NAUTIS Home charts I will be working on.

Paths

Paths are used to create simple or complex shapes combining several straight or curved lines.

The <path> element has one basic attribute that defines the points and other commands of how to draw the path:

AttributeDescription
dRequired. A set of commands which define the path. The following commands are available for path data:
• M = moveto (move from one point to another point)
• L = lineto (create a line)
• H = horizontal lineto (create a horizontal line)
• V = vertical lineto (create a vertical line)
• C = curveto (create a curve)
• S = smooth curveto (create a smooth curve)
• Q = quadratic Bézier curve (create a quadratic Bézier curve)
• T = smooth quadratic Bézier curveto (create a smooth quadratic Bézier curve)
• A = elliptical Arc (create a elliptical arc)
• Z = closepath (close the path)
Note: All of the commands above can also be expressed in lower case. Upper case means absolutely positioned, lower case means relatively positioned.

Python to SVG

After figuring out the basics it was possible for me to write a script in python to make my own SVG files. This script uses the x and y coordinates received from terminal input and converts them to a path, which is then saved as SVG.

def main():
    coordinate_list = []

    print('Please enter your coordinates (x, y) below:')

    while True:
        coordinate = input(f"Coordinate {len(coordinate_list) + 1}: ")
        if coordinate.strip() == "":
            break

        try:
            x_str, y_str = coordinate.strip().split(',')
            x, y = float(x_str), float(y_str)
            coordinate_list.append((x, y))
        except ValueError:
            print("Invalid format. Please use x,y")

    svg_path = convert_to_svg_path(coordinate_list)
    print('SVG Path:', svg_path)

    save_as_svg(svg_path, filename="output.svg")
    print("Saved as 'output.svg'.")

def convert_to_svg_path(coords):
    path = f"M {coords[0][0]} {coords[0][1]}"

    for x, y in coords[1:]:
        path += f" L {x} {y}"
    path += " Z"
    return path

def save_as_svg(path_data, filename="output.svg", width = 500, height = 500):

    svg_content = f"""<svg xmlns="http://www.w3.org/2000/svg" width="{width}" height="{height}" viewBox="0 0 {width} {height}">
    <path d="{path_data}" fill="white" stroke="none" stroke-width="2"/>
    </svg>"""

    with open(filename, "w") as file:
        file.write(svg_content)

if __name__ == '__main__':
    main()

Converting coordinates from an XML source

I integrated the BeautifulSoup4 python library to be able to read a list of coordinates from a separate file. I used a .dae file based on the Janneke dynamesh model I received from the context team to test with.

from bs4 import BeautifulSoup

def main():
    
    with open("Janneke_003.dae") as fp:
        soup = BeautifulSoup(fp, features="xml")

    float_array_tag = soup.find("float_array", {"id": "Dynamesh-POSITION-array"})

    float_values = list(map(float, float_array_tag.text.strip().split()))
    coordinates = [(float_values[i], float_values[i + 1]) for i in range(0, len(float_values), 3)]

    svg_path = convert_to_svg_path(coordinates)

    min_x, min_y, width, height = viewport(coordinates)
    save_as_svg(svg_path, filename="output.svg")
    print("Saved as 'output.svg'.")

def convert_to_svg_path(coords):
    path = f"M {coords[0][0]} {coords[0][1]}"

    for x, y in coords[1:]:
        path += f" L {x} {y}"
    path += " Z"
    return path

def viewport(coords):
    x_coords = [x for x, _ in coords]
    y_coords = [y for _, y in coords]

    min_x = min(x_coords)
    max_x = max(x_coords)
    min_y = min(y_coords)
    max_y = max(y_coords)

    width = max_x - min_x
    height = max_y - min_y

    return min_x, min_y, width, height

def save_as_svg(path_data, filename="output.svg", min_x = -20, min_y = -20, width = 40, height = 40):

    svg_content = f"""<svg xmlns="http://www.w3.org/2000/svg" width="{width}" height="{height}" viewBox="{min_x} {min_y} {width} {height}">
    <path d="{path_data}" fill="white" stroke="none" stroke-width="2"/>
    </svg>"""

    with open(filename, "w") as file:
        file.write(svg_content)

if __name__ == '__main__':
    main()

Integrating Alpha Shapes

In computational geometry, an alpha shape, or α-shape, is a family of piecewise linear simple curves in the Euclidean plane associated with the shape of a finite set of points. They were first defined by Edelsbrunner, Kirkpatrick & Seidel (1983). The alpha-shape associated with a set of points is a generalization of the concept of the convex hull, i.e. every convex hull is an alpha-shape but not every alpha shape is a convex hull.

Using alpha shapes seemed like the best solution to the problems with the SVG render based on my research on different options. I quickly discovered the Alpha Shape Toolbox python library was able to test my theory, so I integrated it into my code:

def convert_to_svg_path(coords, alpha=1.0):
    shape = alphashape.alphashape(coords, alpha)

    if isinstance(shape, Polygon):
        shapes = [shape]
    elif isinstance(shape, MultiPolygon):
        shapes = shape.geoms

    path_data = ""
    for poly in shapes:
        x, y = poly.exterior.coords.xy
        points = list(zip(x, y))
        path_data += f"M {points[0][0]} {points[0][1]}"
        for px, py in points[1:]:
            path_data += f" L {px} {py}"
        path_data += " Z "

    return path_data.strip()
